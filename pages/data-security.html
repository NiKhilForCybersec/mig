Interview Q&A

Q: Explain symmetric vs asymmetric encryption.

"Symmetric uses one key for both encryption and decryption - fast, good for bulk data (AES-256). Challenge is key distribution. Asymmetric uses a key pair - public key encrypts, private key decrypts (RSA, ECC). Slower but solves key distribution. In practice, we use hybrid: asymmetric to exchange a symmetric session key, then symmetric for the data."

Q: How would you implement a DLP program?

"Start with discovery - what sensitive data exists and where? Then classify and label. Define policies based on data classification. Deploy in phases: 1) Audit mode to understand data flows and tune rules. 2) Warn mode to educate users. 3) Block mode for enforcement. I'm currently leading this exact process for Purview Endpoint DLP - key is stakeholder communication and iterative tuning."

Q: What is the difference between data at rest and in transit encryption?

"At rest protects stored data - database encryption (TDE), disk encryption (BitLocker), file-level encryption. Protects against physical theft or unauthorized storage access. In transit protects data moving across networks - TLS 1.2/1.3 for web, IPSec for VPN, SFTP for files. Protects against eavesdropping and MITM attacks. Both are essential - encrypt everywhere."
