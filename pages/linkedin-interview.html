Section 1: Core Security Engineering & NGFW Design

"North-South is traffic entering/leaving the networkâ€”internet traffic, connections to external partners. That's your perimeter defense focus. Q3

"I filter by the source and destination involved in the affected communication. Look for deny actions that weren't there before the change. "Post-incident review. What went wrong? Was the change request incompleteâ€”did it list all affected systems? Did we test in a non-production environment first? Q5

Coca-Cola uses DLP across IT and OT environments. How would you design and tune DLP policies to minimize false positives?

"This is directly relevant to my current roleâ€”I'm leading the migration from Symantec DLP to Microsoft Purview Endpoint DLP, and we're in the enforcement phase right now.

Q6

How would you improve visibility and control using a Privileged Access Management (PAM) solution?

"I'll be honestâ€”I haven't directly administered a PAM solution like CyberArk or BeyondTrust. My experience is on the SIEM integration side: ingesting PAM logs into Sentinel, building detection rules around privileged access patterns, and using PAM session data during incident investigations.

How PAM Improves Visibility:
Without PAM, privileged access is a black boxâ€”someone logs in as admin, does things, logs out, and you have minimal audit trail. PAM gives you: who accessed what system, when, for how long, and exactly what commands they ran (through session recording). This is invaluable for compliance and forensics.

How PAM Improves Control:
Just-in-time accessâ€”privileges granted only when needed, automatically revoked after. Users never know the actual password (credential vaulting), so they can't share it or write it down. Automatic credential rotation means even if a credential is compromised, it changes before the attacker can use it.

From Detection Engineering:
I've built alerts around PAM events: privileged sessions outside business hours, sessions to systems the user doesn't normally access, credential checkouts without corresponding sessions (possible theft), multiple failed authentications (credential stuffing). Follow-up Questions:

"Break-glass is for emergencies when normal access failsâ€”the PAM system is down, or there's an urgent situation where approval workflow can't wait. "Automated rotation is the goalâ€”humans shouldn't be changing privileged passwords manually. For service accounts, rotate frequently (daily or weekly) because there's no human impact. The key is that applications and workflows should retrieve credentials dynamically from the vault, not have them hardcodedâ€”otherwise rotation breaks things. How do you integrate PAM with AD or cloud workloads?

"AD integration typically means PAM managing AD admin accountsâ€”domain admins, server admins.
