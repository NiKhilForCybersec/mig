Q1: How is OT security different from IT security?

"The fundamental difference is priority. In IT, we follow the CIA triad with confidentiality firstâ€”protecting data from unauthorized access. In OT, it's flipped to AIC: Availability, Integrity, then Confidentiality. A bottling plant losing production costs tens of thousands per hour. Safety is actually the top priority above allâ€”a misconfigured PLC could cause physical harm.

This changes everything about how we approach security. We can't just push patches monthlyâ€”OT systems might only get patched annually during planned shutdowns. We can't run aggressive vulnerability scansâ€”they can crash PLCs. We can't install EDR agents on legacy Windows XP systems running HMIs. The lifecycle is also differentâ€”IT assets last 3-5 years, OT assets run for 15-25 years, meaning we're securing systems designed before cybersecurity was a concern."

Q2: Explain the Purdue Model and why it matters for OT security.

"The Purdue Model defines a hierarchical architecture for industrial networks, creating clear boundaries between business systems and physical processes. Level 5 is the enterprise networkâ€”ERP, email, cloud. Level 4 is site business planning. Between Level 4 and 3 is the critical Industrial DMZâ€”this is where we place jump hosts, OT monitoring platforms, and data transfer servers. Level 3 is site operations with SCADA and historians. Level 2 is area control with HMIs. Level 1 is PLCs and controllers. Level 0 is the physical processâ€”sensors, motors, valves.

The security principle is strict: no direct connections between IT (Level 4-5) and OT (Level 0-3). Everything goes through the DMZ with dual firewalls. IT firewall faces IT, OT firewall faces OT, and even if one is compromised, the attacker lands in a hardened DMZ, not production systems. This architecture is the foundation of IEC 62443 zone and conduit requirements."

Q3: How would you handle vulnerability management in an OT environment?

"OT vulnerability management requires a completely different approach than IT. First, discovery is passiveâ€”we use tools like Claroty or Nozomi that monitor network traffic from SPAN ports rather than active scanning, which could crash PLCs.

For remediation, patching is often impossibleâ€”the vendor may not support updates, or we can't take production down. Key OT Terms

PLC: Programmable Logic Controller - executes control logic

HMI: Human-Machine Interface - operator control panel

SCADA: Supervisory Control and Data Acquisition

RTU: Remote Terminal Unit - field device

DCS: Distributed Control System

SIS: Safety Instrumented System - independent safety layer

Historian: Time-series database for process data

MES: Manufacturing Execution System

OPC: Open Platform Communications - data exchange standard

Modbus: Serial/TCP protocol for PLCs

EtherNet/IP: Rockwell industrial Ethernet protocol

Ladder Logic: PLC programming language

Notable OT/ICS Attacks (Know These)

Stuxnet (2010) First known cyberweapon targeting industrial systems. Destroyed Iranian nuclear centrifuges via PLC manipulation.
Ukraine Grid (2015/2016) Russian attacks on Ukrainian power grid. First confirmed cyber attacks causing power outages.
TRITON/TRISIS (2017) Targeted Safety Instrumented Systems at Saudi petrochemical plant. Could have caused physical harm.
Colonial Pipeline (2021) Ransomware attack that shut down major US fuel pipeline. IT attack with OT impact.
Oldsmar Water (2021) Attacker accessed water treatment HMI and increased sodium hydroxide to dangerous levels.
