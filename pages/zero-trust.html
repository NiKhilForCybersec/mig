Core Principles

Verify Explicitly

Authenticate and authorize every access request based on all available data points: identity, location, device, service, data classification, anomalies.

Least Privilege

Limit user access with Just-In-Time (JIT) and Just-Enough-Access (JEA). Risk-based adaptive policies. Data protection.

Assume Breach

Minimize blast radius and segment access. Verify end-to-end encryption. Use analytics to detect threats and improve defenses.

Interview Q&A

Q: What is Zero Trust and why is it important?

"Zero Trust is a security model that eliminates implicit trust - instead of assuming everything inside the network is safe, we verify every access request regardless of where it comes from. The three principles: Verify explicitly (authenticate every request), Least privilege (JIT/JEA access), Assume breach (minimize blast radius). It's important because perimeter-based security fails with cloud, remote work, and sophisticated attackers."

Q: How would you implement Zero Trust in an organization?

"Start with identity - it's the new perimeter. First, MFA for everyone, no exceptions. Then Conditional Access policies - require compliant devices, block legacy auth, implement risk-based policies. Next, device enrollment with compliance checks. Then data protection with classification and DLP. Finally, network - replace VPN with ZTNA, implement microsegmentation. Key: it's a journey, not a destination. Start with quick wins (MFA) and iterate."

Q: What are the six pillars of Zero Trust?

"The six pillars: Identity (users, service accounts - MFA, CA, PIM), Devices (compliance, health attestation, EDR), Data (classification, encryption, DLP), Applications (SSO, shadow IT, permissions), Network (microsegmentation, encryption, ZTNA), Infrastructure (JIT access, telemetry, IaC). All pillars feed into a unified policy decision point that evaluates context for every access request."
