2. IT-OT Boundary Firewall Rules 

┌─────────────────────────────────────────────────────────────────────────────────────┐
│                    IT-OT BOUNDARY FIREWALL DESIGN                                   │
│                  (Defense in Depth at the iDMZ)                                     │
└─────────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ ARCHITECTURE: DUAL FIREWALL                                                         │
└─────────────────────────────────────────────────────────────────────────────────────┘

    IT NETWORK                    iDMZ                      OT NETWORK
    (Level 4-5)                (Level 3.5)                  (Level 0-3)
         │                          │                            │
         │    ┌─────────────┐       │       ┌─────────────┐      │
         │    │             │       │       │             │      │
         └────│ FIREWALL A  │───────┼───────│ FIREWALL B  │──────┘
              │ (IT-facing) │       │       │ (OT-facing) │
              │             │       │       │             │
              └─────────────┘       │       └─────────────┘
                                    │
                          ┌─────────┴─────────┐
                          │                   │
                     ┌────┴────┐         ┌────┴────┐
                     │ Jump    │         │ Data    │
                     │ Server  │         │ Diode/  │
                     │         │         │ Historian│
                     └─────────┘         └─────────┘

    WHY DUAL FIREWALL?
    • Different vendors = different vulnerabilities
    • Compromise of one doesn't compromise both
    • Clear ownership (IT owns A, OT owns B)

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ RULE PRINCIPLES:                                                                    │
└─────────────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ 1. DEFAULT DENY EVERYTHING                                                    │
    │                                                                               │
    │    Last rule: DENY any any any LOG                                           │
    │    Every allowed flow must be explicitly defined                             │
    └───────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ 2. NO DIRECT IT ↔ OT TRAFFIC                                                  │
    │                                                                               │
    │    ❌ WRONG: IT Server → PLC                                                  │
    │    ✅ RIGHT: IT Server → Jump Server (iDMZ) → PLC                            │
    │                                                                               │
    │    Everything terminates in iDMZ                                             │
    └───────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ 3. OT INITIATES TO iDMZ (Not the reverse)                                     │
    │                                                                               │
    │    ❌ WRONG: iDMZ → Poll OT devices                                          │
    │    ✅ RIGHT: OT devices → Push data to iDMZ Historian                        │
    │                                                                               │
    │    Reduces attack surface from IT toward OT                                  │
    └───────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ 4. SPECIFIC PORTS AND PROTOCOLS ONLY                                          │
    │                                                                               │
    │    ❌ WRONG: Allow any port                                                   │
    │    ✅ RIGHT: Allow TCP/443 to specific historian IP only                     │
    │                                                                               │
    │    OT protocols: Modbus (502), DNP3 (20000), OPC-UA (4840)                   │
    └───────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ EXAMPLE RULESET (IT-Facing Firewall A):                                             │
└─────────────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ #  │ Source        │ Dest          │ Port    │ Action │ Profile │ Note       │
    │ ───┼───────────────┼───────────────┼─────────┼────────┼─────────┼────────────│

    │ 4  │ SIEM_Collector│ OT_Sec_Plat   │ 514     │ ALLOW  │ Log     │ Log forward│

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ DETECTION RULES FOR OT:                                                             │
└─────────────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ RULE 1: UNAUTHORIZED CROSS-BOUNDARY TRAFFIC                                   │
    │                                                                               │
    │ // Traffic from IT directly to OT (bypassing iDMZ)                           │
    │ OT_Firewall_Logs                                                             │
    │ | where SourceIP in (IT_Network_Range)                                       │
    │ | where DestIP in (OT_Network_Range)                                         │
    │ | where Action == "allow"  // Should never happen!                           │
    │                                                                               │
    │ Severity: CRITICAL                                                           │
    └───────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ RULE 2: UNUSUAL PROTOCOL IN OT NETWORK                                        │
    │                                                                               │
    │ // IT protocols (RDP, SMB) in OT zones                                       │
    │ OT_Firewall_Logs                                                             │
    │ | where DestPort in (3389, 445, 135, 139)  // IT protocols                   │
    │ | where DestIP in (OT_Network_Range)                                         │
    │                                                                               │
    │ Severity: HIGH (could indicate lateral movement)                             │
    └───────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ RULE 3: OT SECURITY PLATFORM ALERTS                                           │
    │                                                                               │
    │ // Claroty/Nozomi alerts forwarded to Sentinel                               │
    │ Syslog                                                                       │
    │ | where SyslogMessage contains "ALERT"                                       │
    │ | where Computer == "claroty-sensor"                                         │
    │ | parse SyslogMessage with * "severity=" Severity *                         │
    │ | where Severity in ("critical", "high")                                     │
    │                                                                               │
    │ Severity: Based on OT platform severity                                      │
    └───────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────┐
    │ RULE 4: JUMP SERVER SUSPICIOUS ACTIVITY                                       │
    │                                                                               │
    │ // Unusual hours, unusual user, unusual commands                             │
    │ JumpServer_Logs                                                              │
    │ | where TimeGenerated between (time(22:00)..time(06:00))                     │
    │ | where Activity == "login"                                                  │
    │                                                                               │
    │ OR                                                                           │
    │                                                                               │
    │ JumpServer_Logs                                                              │
    │ | where CommandLine contains_any ("powershell", "cmd", "wget", "curl")      │
    │                                                                               │
    │ Severity: MEDIUM-HIGH                                                        │
    └───────────────────────────────────────────────────────────────────────────────┘
