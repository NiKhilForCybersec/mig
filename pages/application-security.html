OWASP Top 10 (2021)

A01

Broken Access Control

Users can act outside their intended permissions. Access controls fail to restrict what authenticated users can do.

‚ùå Examples

  • ‚Ä¢ IDOR: /user/123 ‚Üí /user/456
  • ‚Ä¢ Privilege escalation
  • ‚Ä¢ Missing function-level access

‚úì Prevention

  • ‚Ä¢ Deny by default
  • ‚Ä¢ Server-side validation
  • ‚Ä¢ Rate limiting, log failures
A02

Cryptographic Failures

Failures related to cryptography leading to sensitive data exposure. Previously called "Sensitive Data Exposure".

‚ùå Examples

  • ‚Ä¢ HTTP instead of HTTPS
  • ‚Ä¢ Weak algorithms (MD5, SHA1)
  • ‚Ä¢ Hardcoded keys/passwords

‚úì Prevention

  • ‚Ä¢ TLS 1.2+ everywhere
  • ‚Ä¢ AES-256, bcrypt/Argon2
  • ‚Ä¢ Key management (HSM)
A03

Injection

User-supplied data sent to interpreter as part of command/query. Was #1 for years - still critical!

‚ùå Types

  • ‚Ä¢ SQLi: ' OR 1=1--
  • ‚Ä¢ XSS: <script>alert(1)</script>
  • ‚Ä¢ Command: ; rm -rf /
  • ‚Ä¢ LDAP, XPath, etc.

‚úì Prevention

  • ‚Ä¢ Parameterized queries
  • ‚Ä¢ Input validation (whitelist)
  • ‚Ä¢ Output encoding
  • ‚Ä¢ WAF as defense-in-depth
A04

Insecure Design

Missing or ineffective security controls. Fix: Threat modeling, secure design patterns.

A05

Security Misconfiguration

Default configs, unnecessary features, verbose errors. Fix: Hardening, automation.

A06

Vulnerable Components

Outdated libraries with known CVEs. Fix: SCA scanning, patch management, SBOM.

A07

Auth Failures

Weak passwords, session issues, no MFA. Fix: MFA, secure session management.

A08

Software Integrity

Insecure CI/CD, no code signing. Fix: Verify integrity, secure pipelines.

A09

Logging Failures

No logging, no monitoring, no alerting. Fix: Log security events, SIEM, alerting.

A10

SSRF

Server makes requests to attacker URLs. Fix: Whitelist URLs, segment networks.

Interview Q&A

Q: What is SQL injection and how do you prevent it?

"SQL injection is when user input is included directly in SQL queries, allowing attackers to manipulate the query. Example: ' OR 1=1-- bypasses login. Prevention: Parameterized queries (prepared statements) - never concatenate user input into SQL. Also: input validation (whitelist), least privilege DB accounts, WAF as defense-in-depth."

Q: Explain the difference between SAST and DAST.

"SAST (Static) analyzes source code without running the app - like reviewing blueprints. Finds issues early, gives line numbers, but has false positives. DAST (Dynamic) tests the running application - like stress-testing a building. Finds real vulnerabilities attackers would see, but late in cycle and no code location. Best practice: use both - SAST in dev, DAST in staging."

Q: What is XSS and the types?

"XSS (Cross-Site Scripting) injects malicious scripts into web pages viewed by others. Three types: Reflected - payload in URL, executed immediately (phishing links). Stored - payload saved in DB, executed when page loads (comments, profiles). DOM-based - client-side JavaScript processes untrusted data. Prevention: Output encoding, Content Security Policy (CSP), input validation."
