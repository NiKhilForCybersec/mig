3. DLP Migration â€” Deep Dive

Key DLP Concepts

Sensitive Information Types (SITs):

Pattern-based classifiers that identify sensitive data. Examples: Credit card numbers (regex + checksum), Social Insurance Numbers, custom patterns for internal IDs.

Exact Data Match (EDM):

Hash-based matching against actual sensitive data (e.g., employee database, customer list). More accurate than patterns, requires data upload and hashing.

Sensitivity Labels:

Classification tags (Public, Internal, Confidential, Highly Confidential) that can trigger DLP policies and apply protection (encryption, watermarks).

  • â€¢ Rule-based policies (IF condition THEN action)

7. PAM & Secrets Management â€” Deep Dive

Privileged Access Management (PAM) Concepts

Break-Glass Accounts:

Emergency access accounts for when normal access fails. Heavily audited, alerts on any use, credentials stored securely (often split between multiple people). Should trigger immediate investigation when used.

Just-In-Time (JIT) Access:

Temporary elevation of privileges for specific tasks. User requests access, gets approval, privileges auto-expire after time window (e.g., 4 hours). Reduces standing privileges. Azure PIM is an example.

Session Recording:

Record privileged sessions (RDP, SSH, console) for forensics and audit. Enables playback of exactly what admin did. Some solutions offer keystroke logging.

Credential Vaulting:

No shared passwords. Credentials stored in vault, checked out when needed, automatically rotated. Users never see actual passwordâ€”vault injects it.

  • â€¢ Audit log validation: Ensure PAM logs flow to SIEM, no gaps
  • â€¢ Password changes: Admin password resets, especially on service accounts

Secrets Management in Cloud

Azure Key Vault Best Practices:

10. KQL Detection Examples

Suspicious PowerShell Execution:

DeviceProcessEvents
| where Timestamp > ago(1h)
| where FileName == "powershell.exe" or FileName == "pwsh.exe"
| where ProcessCommandLine has_any ("-enc", "-encodedcommand", "bypass", "hidden", "downloadstring", "iex")
| project Timestamp, DeviceName, AccountName, ProcessCommandLine
