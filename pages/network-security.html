Interview Q&A

Q: Explain the difference between a traditional firewall and NGFW.

"Traditional firewalls work at L3-L4 - they see TCP port 443 but don't know if it's legitimate HTTPS or malware. NGFWs add L7 inspection - they identify actual applications (App-ID), tie traffic to users (User-ID), and integrate IPS, URL filtering, and SSL decryption."

Q: How would you design network segmentation?

"Design segments by trust level: DMZ for internet-facing (heavily monitored), Corporate for users, Server tier for apps (only from corporate), Data tier for databases (only from app servers), Management for admins (separate from everything). Firewalls between each with default-deny. In Azure: NSGs + Azure Firewall."

Q: IDS vs IPS - when use each?

"IDS is passive (alerts only), IPS is active (blocks). Use IDS when: first deploying, tuning rules, high-sensitivity environments. Use IPS when: rules are tuned, known high-confidence threats, production after learning period. Most orgs run IPS but start new signatures in alert-only mode first."
