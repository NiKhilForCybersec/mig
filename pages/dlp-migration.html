Custom Sensitive Information Types

Migrated regex patterns from Symantec Data Identifiers to Purview Custom SITs.

Canadian Social Insurance Number (SIN)

Production

Symantec Regex:

\b[0-9]{3}[-\s]?[0-9]{3}[-\s]?[0-9]{3}\b

Purview SIT XML:

<Pattern confidenceLevel="85">
  <IdMatch idRef="Regex_CA_SIN"/>
  <Match idRef="Keyword_SIN"/>
</Pattern>

Credit Card Number (PCI-DSS)

Production

Symantec (Luhn + Pattern):

\b(?:4[0-9]{12}(?:[0-9]{3})?|5[1-5][0-9]{14})\b
+ Luhn checksum validation

Purview (Built-in):

// Use built-in SIT:
"Credit Card Number"
// Includes Luhn validation

Internal Project Code (Custom)

Custom SIT

Symantec Regex:

\bPROJ-[A-Z]{2}-[0-9]{4,6}\b

Purview Custom SIT PowerShell:

New-DlpSensitiveInformationType
  -Name "Internal Project Code"
  -Regex "\bPROJ-[A-Z]{2}-\d{4,6}\b"
  -Description "Company project IDs"
